# Доступ к боту через подписку на канал

В проект добавлена логика: клиентский функционал бота доступен только после подписки на основной Telegram-канал.

## Что изменено

1. При `/start` бот создаёт/обновляет пользователя и сделку как раньше.
2. После этого бот проверяет подписку на канал.
3. Если пользователь подписан — открывается обычное приветствие, проф-тест и главное меню.
4. Если пользователь не подписан — бот показывает экран доступа с кнопками:
   - `📣 Подписаться на канал`
   - `✅ Я подписался — открыть доступ`
5. Все клиентские разделы дополнительно защищены middleware:
   - `💰 Whale Профит`
   - `🤖 Торговые роботы`
   - `🎁 Бонус`
   - `📞 Связь с менеджером`
   - квиз/проф-тест
   - клиентские сообщения и триггеры
6. Админы из `ADMIN_IDS` проходят без проверки подписки.
7. Если `MAIN_CHANNEL_ID` не задан, проверка подписки автоматически отключается, чтобы бот не заблокировался случайно.

## Важно для Telegram

Чтобы бот мог проверять подписку пользователя через `getChatMember`, добавьте бота администратором в канал.
Полные права не нужны. Достаточно, чтобы бот был админом канала.

## Переменные .env

Проверьте, что в `.env` указаны:

```env
CHANNEL_URL=https://t.me/your_channel_or_invite_link
MAIN_CHANNEL_ID=-1001234567890
```

`CHANNEL_URL` — ссылка, куда ведёт кнопка подписки.  
`MAIN_CHANNEL_ID` — внутренний id канала. Обычно начинается с `-100`.

## Новые файлы

- `app/services/subscription_service.py` — проверка подписки и сообщения доступа.
- `app/bot/middlewares/subscription_gate.py` — middleware, который закрывает клиентский функционал.

## Изменённые файлы

- `app/bot/keyboards/client.py` — добавлена кнопка проверки подписки.
- `app/bot/routers/client/base.py` — `/start` теперь открывает доступ только после подписки.
- `app/bot/routers/__init__.py` — подключён middleware ко всем клиентским роутерам.
